Form Validation

Client side form validation is used to ensure that data submitted via online forms meets the criteria of the servers requirements. Though it should not be used as the only form of security it does help the UX throuh speeding up the process of finding errors.

This basically translates to the want of certain data in a specific format. It provides protections for both the end user and the company's servers.

Some forms of validation look like:

  • required: this makes data forced to be entered.
  • minlength and maxlength: They restrict the amount of characters that can be put into a field.
  • min and max: These work specifically with integers whereas minlength and maxlength work with strings.
  • type: This requires a specific form of data to be accepted, for example, emails, phone numbers, and many more.

There are also DOM API's that are accepted by most browsers to help with validation.

Fetch

This is a basic fetch statment. This first goes out to the website requesting the data available through the link. It then converts the response into a JSON format that can be modified through JavaScript. It then logs the data to the console for developer viewing. This method uses GET requests which does not offer any security.

fetch('http://example.com/movies.json') .then(response => response.json()) .then(data => console.log(data));

To add a security measure, you can use a POST method instead of a GET method.

fetch('https://example.com', { credentials: 'include' });

This is including credentials for the use in a POST request.

Fetch also allows for the pushing of data to a web service and can work with files that include, JSON, BLOB, txt, and many more.
